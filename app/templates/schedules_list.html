Saved schedule snapshots

Each filter keeps a single current snapshot: whenever you save new daily run times on a filter, the previous row for that filter is removed and replaced. Click a row (not Delete) to load Run 1, Run 2, … status for that snapshot’s local times — default day is the log date in the snapshot timezone. The Timezone column is the zone the app used when that row was saved (historical), not necessarily the zone in use after you change .env. Deleting a row here removes the automated schedule for that filter (same as deleting the schedule on the filter page) and clears the snapshot.
